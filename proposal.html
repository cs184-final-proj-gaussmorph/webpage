Final Project Proposal

Zishan Wang, Zeyu Deng, Jack Wang, Owen Zou

Abstract

GaussMorph uses 3D Gaussian Splatting (3DGS) to support object deformation and texture editing. Our approach takes advantage of the strengths of Gaussians to recover fine details that are often lost when using mesh rasterization.

Problem Description

Although object editing methods rely on mesh representations are effective for deformations, they are limited when it comes to expressing complex details such as fuzziness. In contrast, 3D Gaussian Splatting supports fine detail recovery, delivering superior rendering quality by preserving subtle surface features that are often lost in direct mesh rasterization. Our project focuses on harnessing the advantages of 3DGS to build a model that supports object deformation and texture editing. We propose to use mesh data as a geometric guide for deformation and texture edition.

Goals and Deliverbles

The image we will create

We will edit the trained Gaussian point cloud to generate images of the same object with new shapes and new textures—images that do not exist in the original dataset.

Baseline Plan

1. Deformation Support

- Bind Gaussians to the triangular faces of a mesh.

- Utilize barycentric coordinates of each triangle face to represent the position of the associated Gaussian. This enables us to more accurately compute each Gaussian’s position and covariance matrix based on the mesh’s deformation and the local transformation defined in the triangle’s coordinate system.

2. Texture Editing Support

- Use the xatlas library to unwrap the mesh and get vertex UV coordinates.

- Calculate the UV coordinates for each Gaussian by weighting the vertex UVs with the Gaussian’s barycentric coordinates.

- Develop a texture extraction method where we first learn from each Gaussian’s color to create a texture image by gaussians' UV coordinates. Once the texture image is built, users can edit it. The edited texture is then used to update the colors of the Gaussians according to their UV mappings.

3. Integration

- Combine the deformation and retexturing modules into the existing 3DGS framework so that both shape and texture can be edited at the same time.

Aspirational Plan

- Extend the editing framework to support multiple objects or entire scenes.

- Implement an interactive viewer that allows users to explore and manipulate the object in real time.

Performance Metrics

- Editing Quality: Check if the edited objects show natural deformations and texture changes without any artifacts or blurriness.

- Rendering Quality: Measure how well the system recovers fine details, especially for complex surfaces.

- The final deliverable will be a demonstration video showing the evolution of the object under continuous deformation and texture editing.

Schedule

Week 1 Setup

- Study the 3DGS codebase to understand its structure and workflow.

- Set up the development environment.

- Use xatlas to unwrap a mesh and analyze the resulting UV layout.

Week 2 Deformation Development

- Implement deformation calculations to update Gaussian positions and covariance matrices based on mesh transformations.

- Validate the deformation functionality on mesh models.

Week 3 Retexture Module Development

- Develop a method to extract textures from Gaussian colors, creating a texture image.

- Implement texture editing that updates Gaussian colors based on the modified texture.

- Test the retexture process.

Week 4 Integration, Testing, and Pre-demo Preparation

- Integrate the deformation and retexture modules

- Prepare materials for the presentation

- Optionally, implement an interactive viewer for real-time preview.
